Порядок запуска и компиляции проекта.


1)В командной строке(лучше использовать git-bash) выполнить установку всех модулей через команду (npm install)

2)При успешной установке, с имеющими тестовыми файлами проекта(уже есть в данном проекте), протестируйте команды:

    !!!В первую очередь необходимо поднять сервер на backend, команда npm run dev, далее открыть 2 терминал и продолжить запуск по инструкции ниже =>

    2.1)npm run dev (режим разработчика-компиляция без минификации для быстрой разработки приложения)
    2.2)npm run build (режим Production - компиляции файлов и их минификация, сжатие,сонвертация и мн.др(см.plugins.js))
    2.3)npm run zip (создание zip-архива(без node-modules), можно использовать npm run dev или build)
    2.4)npm run deploy (отправка на ftp)- необходимо перед этим настоить все пути и данные для отправки на сервер!

3)Возможно появиться ошибка по плагину webp converter (webp-c ), необходимо его установить через команднкю строку через команду 
                            ***  npm install webp-converter@2.2.3 --save-dev      ****
    3.1) Если так же будет ошибка плагина webp зайдите в каталог node_modules, находите там папку webp-converter-разворачиваете её -
     и из папки src переносите все содержимое в папку-родитель webp-converter.

Процесс кампиляции данных в dist.

1)HTML
    1.1)Глабальные файлы html, которые потом заносятся в папку dist находятся глабально относительно папки src.
    1.2) В данные файлы можно вносить(инклудить(@@include))расширяюшие файлы,которые находятся в src/html/*.html
    1.3)!!!ДЛЯ ПОДКЛЮЧЕНИЯ css необходимо подключать link сразу к каталогу dist/css/*.min.css (например css/index-style.min.css(пример в текущем проекте))

2)SCSS
    2.1)Файлы scss, которые отправляются в папку dist на build, находятся в каталоге /src/scss/global-style/*.scss 
         (можете переименовать global-style и\или вынести файлы глобально,ВАЖНО!, не забудьте поменять название и\или путь в path.js(находиться в config))
    2.2)Файл в папке scss - null.scss(обнуление стилей), можете удалить или перенести.

3)Аналогично с js и др надстройками.





4) ZIP
    4.1) При архивировании проекта, при дальнейшем использовании gulp, необходимо копировать в свой проект только папки gulp,src и package.json,gulpfile!
         и повторить все действия в пункта № 1)  
    4.2) Архивировать проект командой npm run zip   


5) Если при вызове npm run build будет ошибка build paralel<html(к примеру), поменять порядок в task(html),следующим способом.
            с- 
        /*-------------------------------------------------*/   
        .pipe(fileinclude())
        .pipe(app.plugins.replace(/@img\//g, 'img/'))
        .pipe(
            app.plugins.if(
                app.isBuild,
                webpHtmNosvg()
            )
        )
        меняем на-

                 .pipe(
            app.plugins.if(
                app.isBuild,
                webpHtmNosvg()
            )
            .pipe(fileinclude())
            .pipe(app.plugins.replace(/@img\//g, 'img/'))
            /*---------------------------------------------------*/


            То есть, поменять прядок выполнения задачи с fileinclude  на  webpHtmNosvg!
